## Phase 1: MVP (예상 기간: 약 1주)

### 목표
"결정 입력 → AI 3종 분석(장단점/비교표/SWOT) → 결과 보고 본인 결정 저장"의 한 사이클을 본인+지인이 실제로 사용 가능한 수준으로 완성한다.

### 기능
- [ ] 소셜 로그인 (구글, 카카오 — 최소 구글 1개)
- [ ] 결정 입력 폼 (제목, 맥락, 옵션 N개)
- [ ] 장단점 자동 생성 (AI, 결정 전체 + 옵션별)
- [ ] 옵션 비교표 + 가중치 점수
  - 사용자가 비교 기준 추가 (예: 연봉, 출퇴근 시간, 성장성)
  - 가중치 1-5 슬라이더
  - AI가 옵션별 점수 매김 + 가중 합산 자동 계산
- [ ] SWOT 분석 (결정 전체 또는 선택한 옵션)
- [ ] AI 톤 조절 (논리/감성/냉철 — 드롭다운으로 재분석)
- [ ] 결정 히스토리 (목록 조회, 상세 보기, 검색)
- [ ] 본인 최종 결정 저장 (선택 옵션 + 한 줄 메모)
- [ ] 모바일 반응형 (Tailwind responsive)

### 데이터
- User
- Decision
- Option
- Criteria
- Score
- ProsConItem
- SwotItem

### 인증
- Supabase Auth (구글 OAuth)
- 모든 테이블에 RLS 정책 (`user_id == auth.uid()`)

### "진짜 제품" 체크리스트
- [ ] 실제 Supabase DB 연결 (목업 데이터 X)
- [ ] 실제 Google OAuth (하드코딩된 비밀번호 X)
- [ ] Vercel에 배포되어 URL로 접속 가능 (localhost X)
- [ ] 본인+지인 5명 이상이 URL로 접속해서 결정 1건 이상 입력
- [ ] Anthropic Claude API 실호출 (LLM 응답 페이크 X)
- [ ] 모바일 사파리/크롬에서 동작 검증

## Phase 2: 확장 (예상 기간: 1-2주, Phase 1 안정화 후)

### 전제 조건
- Phase 1이 Vercel에 배포되어 본인+지인이 실제로 사용 중
- 1주일 재방문율 데이터 확보
- 기능 추가 우선순위 확인 (히스토리 잘 안 보면 회고 알림 우선)

### 목표
"결정 후 회고"라는 재방문 루프를 만들어서 1회성 사용 → 누적 사용으로 전환.

### 기능
- [ ] 결정 회고 화면 ("그 결정 어땠나요?" 1-5점 + 자유 메모)
- [ ] 회고 알림 (1주 후 / 1개월 후) — 메일 또는 푸시
- [ ] PWA 설치 가능 (manifest.json, service worker, 홈 화면 추가)
- [ ] 푸시 알림 권한 요청 + 발송 (Web Push API)
- [ ] 결정 링크 공유 (비공개 토큰 URL)
- [ ] 자주 쓰는 비교 기준 템플릿 ("이직 기준 세트", "구매 기준 세트" 저장/불러오기)

### 추가 데이터
- Review 테이블 추가
- Decision에 share_token 필드 추가
- CriteriaTemplate 테이블 (기준 세트 저장)

### 통합 테스트
- Phase 1 핵심 흐름 (결정 입력 → 분석 → 저장)이 여전히 동작하는지
- Supabase 마이그레이션이 기존 데이터 손상 없이 진행되는지
- PWA 설치 후 오프라인에서 히스토리 조회는 가능 (분석은 온라인 필요)

## Phase 3: 고도화 (예상 기간: 2-4주, Phase 2 정착 후)

### 전제 조건
- Phase 1 + 2가 안정 운영 (3개월 이상)
- 누적 결정 데이터 100건+ (패턴 분석 의미 있음)
- 본인+지인 외 일반 사용자 유입 시작

### 목표
개인 결정 도우미를 넘어, 누적 데이터로 사용자에게 "당신은 이런 패턴이네요" 인사이트 제공 + 협업 결정으로 사용 시나리오 확장.

### 기능
- [ ] 의사결정 패턴 분석 대시보드 (예: "안전한 옵션 60%, 도전 40%", "후회한 결정 vs 만족한 결정 비교")
- [ ] What-if 시뮬레이션 (가중치 슬라이더로 실시간 결과 변화)
- [ ] 친구/가족 협업 결정 (한 결정에 여러 명 점수 매김, 평균/투표 표시)
- [ ] 음성 입력 (Web Speech API)
- [ ] Notion/Obsidian 내보내기 (마크다운 export, Notion API)

### 주의사항
- 협업 결정: 동시 편집 충돌 처리 (Supabase Realtime 또는 단순 폴링)
- What-if: 점수 재계산이 클라이언트에서 즉시 반영되어야 함 (서버 왕복 X)
- 패턴 분석: AI 호출 비용 — 사용자별로 캐싱하고 주간 1회 갱신
- Notion API: OAuth 인증 흐름 추가 부담
