## Phase 1: MVP (예상 기간: 약 1주)

### 목표
"결정 입력 → AI 3종 분석(장단점/비교표/SWOT) → 결과 보고 본인 결정 저장"의 한 사이클을 본인+지인이 실제로 사용 가능한 수준으로 완성한다.

### 기능
- [ ] 소셜 로그인 (구글, 카카오 — 최소 구글 1개)
- [ ] 결정 입력 폼 (제목, 맥락, 옵션 N개)
- [ ] 장단점 자동 생성 (AI, 결정 전체 + 옵션별)
- [ ] 옵션 비교표 + 가중치 점수
  - 사용자가 비교 기준 추가 (예: 연봉, 출퇴근 시간, 성장성)
  - 가중치 1-5 슬라이더
  - AI가 옵션별 점수 매김 + 가중 합산 자동 계산
- [ ] SWOT 분석 (결정 전체 또는 선택한 옵션)
- [ ] AI 톤 조절 (논리/감성/냉철 — 드롭다운으로 재분석)
- [ ] 결정 히스토리 (목록 조회, 상세 보기, 검색)
- [ ] 본인 최종 결정 저장 (선택 옵션 + 한 줄 메모)
- [ ] 모바일 반응형 (Tailwind responsive)

### 데이터
- User
- Decision
- Option
- Criteria
- Score
- ProsConItem
- SwotItem

### 인증
- Supabase Auth (구글 OAuth)
- 모든 테이블에 RLS 정책 (`user_id == auth.uid()`)

### "진짜 제품" 체크리스트
- [ ] 실제 Supabase DB 연결 (목업 데이터 X)
- [ ] 실제 Google OAuth (하드코딩된 비밀번호 X)
- [ ] Vercel에 배포되어 URL로 접속 가능 (localhost X)
- [ ] 본인+지인 5명 이상이 URL로 접속해서 결정 1건 이상 입력
- [ ] Anthropic Claude API 실호출 (LLM 응답 페이크 X)
- [ ] 모바일 사파리/크롬에서 동작 검증

## Phase 2: 확장 (예상 기간: 1-2주, Phase 1 안정화 후)

> **2026-05-08 갱신:** Phase 2 범위와 기능을 [06_PHASE2_PRD.md](./06_PHASE2_PRD.md)로 상세화함. 이 섹션은 요약만 유지.

### 전제 조건
- Phase 1이 Vercel에 배포되어 본인+지인이 실제로 사용 중 ✅ (2026-05-08 배포 완료)
- ~~1주일 재방문율 데이터 확보~~ — 이 전제에 묶이는 기능은 Phase 3로 이동, 데이터 무관 기능만 Phase 2 유지

### 목표
Phase 1의 한 사이클(결정 입력 → 분석 → 저장)에 **분석 깊이**(탭 5종 + 추천 점수)와 **재방문 루프**(회고 + 알림)를 더해서, 1회성 도구를 **누적되는 의사결정 동반자**로 만든다.

### 기능 (요약 — 상세는 [06_PHASE2_PRD.md](./06_PHASE2_PRD.md))
- [ ] **분석 탭 5종 확장** (장단점/비교표/SWOT 유지 + **데카르트 4분면** + **피터 드러커 5질문**)
- [ ] **추천 점수** (옵션 상단, 0–100, 색상 구분, 신뢰도 라벨)
- [ ] **사용자 경향성 추적** (5개 차원 + 만족 보정값, EWMA 업데이트)
- [ ] 결정 회고 화면 (1-5점 + 자유 메모)
- [ ] 회고 알림 (1주 / 1개월, Web Push + 이메일 fallback)
- [ ] 결정 링크 공유 (비공개 토큰, 읽기 전용)
- [ ] 비교 기준 템플릿 (시스템 기본 3종 + 사용자 정의)
- [ ] **PWA** (manifest + service worker + 홈 화면 추가) — *모바일 앱 A안*

### 추가 데이터 (마이그레이션 0003)
- 신규: `CartesianItem`, `DruckerAnswer`, `UserTendency`, `RecommendationScore`, `CriteriaTemplate`, `Review` 테이블
- Decision에 `share_token` 필드, Option에 `ai_labels JSONB` 필드 추가

### 통합 테스트
- Phase 1 핵심 흐름 (결정 입력 → 분석 → 저장)이 여전히 동작하는지
- Supabase 마이그레이션 0003이 기존 데이터 손상 없이 진행되는지
- PWA 설치 후 오프라인에서 히스토리 조회는 가능 (분석은 온라인 필요)
- 추천 점수가 신뢰도 라벨(룰 기반/학습 중/경향성 반영) 단계별로 다르게 산정되는지

## Phase 3: 고도화 (예상 기간: 2-4주, Phase 2 정착 후)

### 전제 조건
- Phase 1 + 2가 안정 운영 (3개월 이상)
- 누적 결정 데이터 100건+ (패턴 분석 의미 있음)
- 1주일+ 재방문율 데이터 확보 (Phase 2에서 미룬 데이터 의존 기능 시작 시점)
- 본인+지인 외 일반 사용자 유입 시작

### 목표
개인 결정 도우미를 넘어, 누적 데이터로 사용자에게 "당신은 이런 패턴이네요" 인사이트 제공 + 협업 결정으로 사용 시나리오 확장.

### 기능
- [ ] 의사결정 패턴 분석 대시보드 (예: "안전한 옵션 60%, 도전 40%", "후회한 결정 vs 만족한 결정 비교")
  - Phase 2의 UserTendency 5차원을 풀 인사이트로 확장
- [ ] **회고 텍스트 분석** — Review.reflection의 후회/만족 키워드를 경향성 보정에 반영 (Phase 2에서 미룬 항목)
- [ ] What-if 시뮬레이션 (가중치 슬라이더로 실시간 결과 변화)
- [ ] 친구/가족 협업 결정 (한 결정에 여러 명 점수 매김, 평균/투표 표시)
- [ ] 음성 입력 (Web Speech API)
- [ ] Notion/Obsidian 내보내기 (마크다운 export, Notion API)
- [ ] **사용자 정의 경향성 차원** (Phase 2의 5차원 외에 사용자가 직접 추가)

### 주의사항
- 협업 결정: 동시 편집 충돌 처리 (Supabase Realtime 또는 단순 폴링)
- What-if: 점수 재계산이 클라이언트에서 즉시 반영되어야 함 (서버 왕복 X)
- 패턴 분석: AI 호출 비용 — 사용자별로 캐싱하고 주간 1회 갱신
- Notion API: OAuth 인증 흐름 추가 부담

## Phase 4: 네이티브 앱 (예상 기간: 4-8주, Phase 2 PWA 정착 후)

### 전제 조건
- Phase 2 PWA가 안정 운영 (3개월 이상)
- PWA 설치 사용자 50명+ 또는 "스토어에서 검색해서 깔고 싶다"는 요청 3건+
- 백엔드(Supabase + API)는 그대로 공유, 클라이언트만 새로 구축

### 목표
앱 스토어 검색 가능성과 네이티브 UX(위젯, 푸시 사운드, 생체 인증)가 필요한 사용자에게 도달.

### 기능
- [ ] React Native(또는 Expo)로 iOS/Android 네이티브 앱
- [ ] App Store + Play Store 등록
- [ ] 네이티브 푸시 알림 (FCM/APNs)
- [ ] 위젯 (홈 화면 "오늘 결정할 것" 위젯)
- [ ] Face ID / Touch ID 잠금 (앱 진입 시)
- [ ] 다크모드 시스템 연동

### 주의사항
- 백엔드는 Phase 2와 공유 — Supabase 클라이언트가 RN에서 동작
- Google OAuth는 RN에서 별도 SDK 필요 (Supabase Auth + Expo AuthSession)
- 결제/심사: 무료 앱이라 인앱 결제는 없지만 심사 시 "AI 사용" 명시 필요
