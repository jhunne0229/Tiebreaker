# 타이브레이커(Tiebreaker) -- PRD (Product Requirements Document)

> 생성일: 2026-05-07
> 생성 도구: Show Me The PRD

---

## 1. 제품 개요

### 한 줄 요약
결정 못 내릴 때 AI가 장단점·비교표·SWOT을 한 번에 보여줘서 판단을 돕는 개인용 의사결정 도우미.

### 해결하는 문제
- 일상의 사소한 선택부터 이직·이사·진로 같은 큰 결정까지, 머릿속에서만 굴리면 같은 생각이 반복되고 결론이 안 난다.
- 친구에게 물어보기엔 정보를 다시 설명해야 하고, 답이 친구의 편견에 좌우된다.
- ChatGPT에 물어볼 수 있지만 매번 "장단점 정리해줘 / 표로 비교해줘" 같은 프롬프트를 짜야 하고 기록이 흩어진다.

### 핵심 가치
- **한 번 입력으로 3종 분석**: 자연어로 결정 상황만 적으면 장단점 / 비교표 / SWOT 결과를 동시에 받는다 (개별 프롬프트 불필요).
- **AI 톤 조절**: 같은 결정을 "논리적", "감성적", "냉철한 친구" 톤으로 다르게 분석 가능 → 다각도 검토.
- **결정 히스토리**: 흩어진 채팅이 아니라 "결정"이라는 단위로 누적 → 나중에 비슷한 결정 시 참고.
- **무난한 PWA**: 별도 설치 없이 모바일에서도 앱처럼 동작.

### 경쟁 서비스 차별점
- Rationale.ai: 영문 위주, 결정 누적/히스토리 약함 → 우리는 한글 + 히스토리 강조
- ChatGPT 직접 사용: 매번 프롬프트 재작성 → 우리는 결정 입력 → 자동 3종 분석

## 2. 사용자

### 주요 사용자
- **누구**: 본인 + 지인 (개인 사용자, 결정을 어려워하거나 정리해서 판단하고 싶어하는 사람)
- **상황**:
  - 일상: 점심 메뉴, 어떤 노트북 살지, 이 사람과 만날지 말지
  - 큰 결정: 이직 제안 받았을 때, 자취 시작할지, 대학원 갈지
- **목표**: AI에게 다양한 관점을 받아서 본인 판단의 보조 자료로 쓰기 (결정 자체는 본인이)

### 사용자 시나리오

**시나리오 A — 일상 결정**
1. 사용자가 "회사 근처 원룸 vs 본가에서 통근" 고민이 생긴다
2. 타이브레이커를 열고 결정 제목과 두 옵션을 입력한다
3. AI가 자동으로 장단점, 비교표(가격/시간/스트레스 등 기준), SWOT을 생성한다
4. 사용자는 "냉철한 친구" 톤으로 다시 보기를 누른다
5. 본인의 결정을 적고 저장한다 (히스토리에 누적)

**시나리오 B — 큰 결정**
1. 사용자가 이직 제안을 받았다 ("현재 회사 vs 새 회사")
2. 결정 입력 시 비교 기준에 가중치를 준다 (연봉 5점, 출퇴근 3점, 성장성 4점)
3. AI가 옵션별로 기준 점수를 매기고 가중 합산 결과를 보여준다
4. SWOT 분석에서 "위협" 항목을 보고 미처 고려 못한 점을 발견한다
5. 결정 후 1주/1개월 후 회고 알림을 받는다 (Phase 2)

| 장단점 자동 생성 | 입력된 결정/옵션별로 AI가 pros/cons 리스트 생성 | P1 (MVP) | 간단 |

| 회고 알림 (메일/푸시) | 결정 후 일정 시간 뒤 알림 | P2 | 보통 |

| 의사결정 패턴 분석 대시보드 | 누적 결정에서 사용자 성향 분석 | P3 | 복잡 |

| 친구/가족 협업 결정 | 여러 명이 같이 한 결정에 점수 매김 | P3 | 복잡 |
| 음성 입력 | 결정 내용 음성으로 받아쓰기 | P3 | 보통 |
| Notion/Obsidian 내보내기 | 마크다운/API 내보내기 | P3 | 간단 |

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름 (Phase 1)
```
로그인 -> 새 결정 만들기 -> 결정 제목/맥락/옵션 입력
       -> [AI 분석] 장단점 + 비교표 + SWOT 자동 생성
       -> 톤 변경 / 기준 추가 / 가중치 조절
       -> 본인 결정 저장 -> 히스토리 누적
```

### 상세 흐름

1. **로그인**: 구글/카카오로 1초 로그인 (소셜 로그인)
2. **새 결정 만들기**: "+" 버튼 → 빈 결정 폼
3. **결정 입력**:
   - 제목 (예: "이직할까 말까")
   - 맥락 (자연어, 예: "현재 연봉 5천, 새 회사 6천이지만 출퇴근 1시간 늘어남...")
   - 옵션 추가 (예: "현재 회사 유지" / "새 회사 이직") — 1개여도 OK
4. **AI 분석**: "분석하기" 버튼 → 장단점/비교표/SWOT 동시 생성 (탭 UI)
5. **세부 조정**:
   - 톤 변경 드롭다운 (논리/감성/냉철)
   - 비교 기준 직접 추가 + 가중치 슬라이더 (1-5)
   - "다시 분석" 버튼
6. **본인 결정 저장**: 최종 선택 옵션 + 한 줄 메모
7. **히스토리**: 메인 화면에 결정 카드 목록 (제목/날짜/최종선택)

## 5. 성공 기준 (Phase 1 종료 시점)

- [ ] 사용자가 결정 입력 → 분석 결과까지 30초 안에 도달
- [ ] AI 응답 시간 평균 10초 이하 (스트리밍으로 체감 단축)
- [ ] 모바일 사파리/크롬에서 깨짐 없이 동작
- [ ] 본인 + 지인 5명 이상이 실제 결정 1건 이상 입력해서 사용
- [ ] 결정 1건당 평균 분석 횟수 2회 이상 (톤 변경/기준 추가 등 재분석 사용 증명)
- [ ] 1주일 재방문율 30% 이상

## 6. 안 만드는 것 (Out of Scope, Phase 1)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다. AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- 회고 알림 시스템 — 이유: 메일/푸시 인프라 세팅 부담, Phase 2로
- PWA 설치 프롬프트 — 이유: 반응형 웹으로 시작, PWA 매니페스트는 Phase 2
- 결정 공유 링크 — 이유: 권한/익명 처리 복잡, Phase 2
- 협업 결정 (여러 명이 점수 매김) — 이유: 핵심 가치 검증 후 Phase 3
- 결제/구독 — 이유: 본인+지인용 MVP, 무료
- 관리자 대시보드 — 이유: 사용자 < 10명, Supabase 콘솔로 충분
- 다크모드 — 이유: 핵심 가치와 무관, Phase 2 이후
- 다국어 — 이유: 본인+지인 한국어, 나중에
- 음성 입력 — 이유: Web Speech API 기기별 호환 검증 부담, Phase 3
- 노션/옵시디언 연동 — 이유: 인증 흐름 복잡, Phase 3

## 7. [NEEDS CLARIFICATION]

> 아직 결정되지 않은 사항. 개발 전에 정해야 합니다.

- [ ] AI 비용 한도 — Claude API 호출당 비용을 사용자별로 제한할지, 무제한일지 (지인용이라 무제한도 가능하지만 안전장치 필요)
- [ ] 무료 사용자 일일 분석 횟수 제한 (예: 하루 20회) — Phase 2 공유 시작 시점에 결정
- [ ] 결정 데이터 삭제 정책 — 사용자가 회원 탈퇴 시 결정 데이터 즉시 삭제 vs 30일 유예
- [ ] AI 응답 언어 — 한국어 고정 vs 사용자 입력 언어 자동 감지
- [ ] 분석 결과 캐싱 — 동일 결정에 대해 같은 톤이면 캐싱할지 (비용 절감 vs 신선도)
- [ ] 소셜 로그인 제공자 — 구글만 vs 구글+카카오 둘 다
